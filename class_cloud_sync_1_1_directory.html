Detailed Description

Provider-independent representation of a directory.

Can be seen as a pointer to the actual folder in the cloud. This does not hold the contents of the folder in memory but makes a network call each time you ask for a resource.

change directory

Note
The provided path will always be handled as relative path. Leading slashes will be ignored. If you need to pass an absolute path you should do this:
cloud->root()->get_directory("some/absolute/path");
Parameters

  • foldername
  • path/to/other/folder

  • subfolder/.. returns the current folder
  • foldername/ leading slashes are ignored. Same as: foldername
